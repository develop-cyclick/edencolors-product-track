## 1. ภาพรวมระบบ

ระบบ QR Authenticity & Activation ช่วยติดตามและยืนยันความแท้ของสินค้าด้วย QR Code + Serial 12 หลัก

### Flow การทำงานหลัก

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│  Pre-Generate   │ → │   รับเข้าคลัง    │ → │    ส่งออก       │
│   QR Codes      │   │     (GRN)       │   │   (Outbound)    │
└─────────────────┘    └─────────────────┘    └────────┬────────┘
                                                       │
                       ┌─────────────────┐    ┌────────▼────────┐
                       │    Activate     │ ← │ Manager อนุมัติ  │
                       │   (ลูกค้า)       │   │                 │
                       └─────────────────┘    └─────────────────┘
```

### บทบาทผู้ใช้งาน

| Role | สิทธิ์ |
|------|-------|
| **Admin** | จัดการผู้ใช้, คลินิก, Product Masters, Master Data, PO, Event Logs, Settings |
| **Manager** | อนุมัติ/ปฏิเสธใบส่งออก |
| **Warehouse** | สร้าง QR, GRN, Outbound, พิมพ์ Label, Reprint, Return |
| **Public** | สแกนตรวจสอบ, ลงทะเบียนสินค้า |

## 3. สำหรับพนักงานคลัง (Warehouse)

### 3.1 สร้าง QR ล่วงหน้า (Pre-Generate)

**เมนู:** Dashboard → สร้าง QR ล่วงหน้า

**ใช้เมื่อ:**
- ต้องการเตรียม QR Code ก่อนรับสินค้าจริง
- ต้องการพิมพ์ Label ล่วงหน้า

**ขั้นตอน:**
1. ระบุจำนวนที่ต้องการสร้าง (เช่น 100 ชิ้น)
2. กรอกหมายเหตุ (ไม่บังคับ)
3. กดปุ่ม "สร้าง QR"

**ผลลัพธ์:**
- ระบบสร้าง Serial 12 หลักและ QR Token
- สินค้ามีสถานะ `PENDING_LINK` (รอเชื่อมโยง)
- สามารถพิมพ์ Label ได้ทันที

---

### 3.2 สร้างใบรับเข้าคลัง (GRN)

**เมนู:** Dashboard → ใบรับเข้าคลัง → สร้างใหม่

**ขั้นตอน:**
1. **กรอกข้อมูลหัวเอกสาร:**
   - วันที่รับ
   - คลังสินค้า
   - เลขที่ PO (ถ้ามี)
   - ข้อมูลผู้ส่ง (ชื่อ, ที่อยู่, เบอร์โทร)

2. **เพิ่มรายการสินค้า:**
   - **วิธีที่ 1:** สแกน QR จาก Pre-Generated Batch
   - **วิธีที่ 2:** กรอกข้อมูลสินค้าเอง

   ข้อมูลที่ต้องกรอก:
   - Serial 12 หลัก (จาก Pre-Gen หรือระบบสร้างให้)
   - หมวดหมู่สินค้า
   - SKU, ชื่อสินค้า, ขนาด/รุ่น
   - Lot, วันผลิต, วันหมดอายุ
   - สถานะตรวจรับ (OK/เสียหาย/เคลม)

3. กดปุ่ม "บันทึก"

**ผลลัพธ์:**
- ระบบสร้างเลข GRN อัตโนมัติ (GRN-2026-000001)
- สินค้าที่เป็น `PENDING_LINK` เปลี่ยนเป็น `IN_STOCK`
- สินค้าใหม่มีสถานะ `IN_STOCK`

---

### 3.3 สร้างใบส่งออก (Outbound)

**เมนู:** Dashboard → ใบส่งออก → สร้างใหม่

**ขั้นตอน:**
1. **กรอกข้อมูลหัวเอกสาร:**
   - คลังต้นทาง
   - คลินิกปลายทาง
   - วิธีจัดส่ง
   - ข้อมูลติดต่อ
   - อ้างอิง PO (ถ้ามี)

2. **เลือกสินค้าที่จะส่ง:**
   - เฉพาะสินค้าสถานะ `IN_STOCK`
   - สามารถค้นหาด้วย Serial หรือ SKU

3. **เลือก Action:**
   - **บันทึกร่าง:** เก็บไว้แก้ไขภายหลัง
   - **ส่งขออนุมัติ:** ส่งให้ Manager อนุมัติ

**ผลลัพธ์:**
- ระบบสร้างเลข OUT อัตโนมัติ (OUT-2026-000001)
- ถ้าส่งขออนุมัติ: สินค้าเปลี่ยนเป็น `PENDING_OUT`
- รอ Manager อนุมัติ

---

### 3.4 พิมพ์ QR Label

**เมนู:** Dashboard → ใบรับเข้าคลัง/ใบส่งออก → พิมพ์ Label

**ขั้นตอน:**
1. เปิดรายละเอียด GRN หรือ Pre-Gen Batch
2. เลือกสินค้าที่ต้องการพิมพ์
3. กดปุ่ม "พิมพ์ Label"
4. ระบบ Download ไฟล์ PDF

**รูปแบบ Label:**
- ขนาด 4x6 นิ้ว (101.6 x 152.4 mm)
- 1 หน้า = 1 Serial
- เนื้อหา: QR Code + Serial 12 หลัก + ข้อมูลสินค้า

**การพิมพ์:**
- ใช้โหมด Actual size / 100%
- ห้ามใช้ Fit to page

---

### 3.5 พิมพ์ QR ใหม่ (Reprint)

**เมนู:** Dashboard → พิมพ์ QR ใหม่

**ใช้เมื่อ:**
- Label เดิมชำรุด/เปื้อน
- สินค้าติด Label ผิด

**ขั้นตอน:**
1. ค้นหาสินค้าด้วย Serial Number
2. เลือกสินค้าที่ต้องการพิมพ์ใหม่
3. ระบุเหตุผล
4. กดปุ่ม "พิมพ์ใหม่"

**ผลลัพธ์:**
- Token เดิมถูก Revoke (ใช้ไม่ได้แล้ว)
- สร้าง Token ใหม่ (Version +1)
- Download PDF Label ใหม่

**ข้อจำกัด:**
- ไม่สามารถ Reprint สินค้าที่ `ACTIVATED` แล้ว
- ไม่สามารถ Reprint สินค้าที่ `RETURNED` แล้ว

---

### 3.6 รับคืนสินค้า (Return)

**เมนู:** Dashboard → รับคืนสินค้า

**ขั้นตอน:**
1. ค้นหาสินค้าด้วย Serial Number
2. เลือกสินค้าที่จะรับคืน
3. เลือกเหตุผลการคืน
4. กรอกหมายเหตุ (ถ้ามี)
5. กดปุ่ม "บันทึกการคืน"

**ผลลัพธ์:**
- สินค้าเปลี่ยนสถานะเป็น `RETURNED`
- บันทึก Event Log

**ข้อจำกัด:**
- รับคืนได้เฉพาะสถานะ `SHIPPED` หรือ `ACTIVATED`

---

### 3.7 จัดการสินค้าเสียหาย (Damaged)

**เมนู:** Dashboard → สินค้าเสียหาย

**ขั้นตอน:**
1. ค้นหาสินค้าที่เสียหาย
2. ระบุเหตุผล (แตก/บุบ/หมดอายุ)
3. กดปุ่ม "บันทึก"

**ผลลัพธ์:**
- สินค้าเปลี่ยนสถานะเป็น `DAMAGED`
- บันทึก Event Log

## 4. สำหรับผู้จัดการ (Manager)

### 4.1 อนุมัติ/ปฏิเสธใบส่งออก

**เมนู:** Dashboard → อนุมัติใบส่งออก

**ขั้นตอน:**
1. ดูรายการใบส่งออกที่รอ (`PENDING`)
2. คลิกดูรายละเอียด
3. ตรวจสอบข้อมูล:
   - คลินิกปลายทาง
   - รายการสินค้า
   - ข้อมูลการจัดส่ง
4. เลือก:
   - **อนุมัติ:** สินค้าเปลี่ยนเป็น `SHIPPED`
   - **ปฏิเสธ:** ต้องระบุเหตุผล, สินค้ากลับเป็น `IN_STOCK`

### 4.2 ดูประวัติการอนุมัติ

**Tabs:**
| Tab | แสดงรายการ |
|-----|-----------|
| รออนุมัติ | PENDING |
| อนุมัติแล้ว | APPROVED |
| ปฏิเสธ | REJECTED |

## 5. สำหรับผู้ดูแลระบบ (Admin)

### 5.1 จัดการผู้ใช้

**เมนู:** Dashboard → จัดการผู้ใช้

**สามารถทำได้:**
- เพิ่มผู้ใช้ใหม่
- แก้ไขข้อมูลผู้ใช้
- เปลี่ยน Role
- Reset Password
- ปิดใช้งาน (Disable)

---

### 5.2 จัดการคลินิก

**เมนู:** Dashboard → จัดการคลินิก

**ข้อมูลคลินิก:**
- ชื่อคลินิก / สาขา
- จังหวัด
- สถานะ Active/Inactive

**ฟีเจอร์เพิ่มเติม:**
- ดู Reservation (สินค้าที่สั่งฝาก)

---

### 5.3 Product Masters (ใหม่)

**เมนู:** Dashboard → แคตาล็อกสินค้า

**ข้อมูล Product Master:**
- SKU, ชื่อ (TH/EN)
- รูปภาพ
- หมวดหมู่
- ขนาด/รุ่น
- ประเภท Activation (SINGLE/PACK)
- จำนวน Activation สูงสุด (สำหรับ PACK)

**การใช้งาน:**
- เป็น Template สำหรับสร้าง ProductItem
- กำหนดว่าสินค้านี้ Activate ได้กี่ครั้ง

---

### 5.4 Purchase Orders (ใหม่)

**เมนู:** Dashboard → ใบสั่งซื้อ

**ใช้สำหรับ:**
- บันทึกคำสั่งซื้อจากคลินิก
- ติดตามการส่งสินค้า

**ขั้นตอน:**
1. สร้าง PO ใหม่
2. เลือกคลินิก
3. เพิ่มรายการสินค้า (จาก Product Master)
4. ยืนยัน PO

**สถานะ PO:**
| สถานะ | ความหมาย |
|-------|----------|
| DRAFT | ร่าง |
| CONFIRMED | ยืนยันแล้ว |
| PARTIAL | ส่งบางส่วน |
| COMPLETED | ส่งครบแล้ว |
| CANCELLED | ยกเลิก |

---

| ประเภท | คำอธิบาย |

## 6. สำหรับลูกค้า/คลินิก (Public)

### 6.1 ตรวจสอบความแท้ (Verify)

**วิธีที่ 1: สแกน QR Code**
1. ใช้กล้องมือถือสแกน QR บนสินค้า
2. ระบบเปิดหน้า Verify อัตโนมัติ

**วิธีที่ 2: พิมพ์ Serial**
1. เข้าเว็บไซต์ `/th/verify` หรือ `/en/verify`
2. กรอก Serial 12 หลัก
3. กดปุ่ม "ตรวจสอบ"

**ผลลัพธ์ที่เป็นไปได้:**

| สถานะ | สี | ความหมาย |
|-------|-----|----------|
| GENUINE_IN_STOCK | เขียว | ของแท้ - อยู่ในคลัง |
| GENUINE_SHIPPED | เขียว | ของแท้ - ส่งออกแล้ว |
| GENUINE_ACTIVATED | เหลือง | ของแท้ - ถูก Activate แล้ว |
| GENUINE_RETURNED | เหลือง | ของแท้ - ถูกส่งคืนแล้ว |
| TOKEN_REVOKED | แดง | Token ถูกยกเลิก (มี QR ใหม่) |
| INVALID | แดง | ไม่พบในระบบ / ของปลอม |

**ข้อมูลที่แสดง:**
- Serial Number
- ชื่อสินค้า / SKU
- หมวดหมู่
- วันหมดอายุ
- สถานะปัจจุบัน
- ข้อมูลคลินิก (ถ้าเปิด setting)
- ข้อมูล Activation (ถ้ามี)

---

### 6.2 ลงทะเบียนสินค้า (Activate)

**URL:** หลังจากสแกน QR → กดปุ่ม "ลงทะเบียน"

**ข้อมูลที่ต้องกรอก:**

| ฟิลด์ | บังคับ |
|-------|--------|
| ชื่อ-นามสกุล | ไม่บังคับ* |
| อายุ | ไม่บังคับ* |
| เพศ | ไม่บังคับ* |
| จังหวัด | ไม่บังคับ* |
| เบอร์โทรศัพท์ | ไม่บังคับ |
| ยินยอม PDPA | **บังคับ** |

*ขึ้นอยู่กับ System Settings

**ประเภทการ Activate:**

| ประเภท | ลักษณะ |
|--------|--------|
| **SINGLE** | Activate ได้ครั้งเดียว แล้วล็อกถาวร |
| **PACK** | Activate ได้หลายครั้ง (ตามที่กำหนด) |

**ข้อจำกัด:**
- ต้องเป็นสินค้าสถานะ `SHIPPED` เท่านั้น
- ต้องติ๊กยินยอม PDPA

**ผลลัพธ์:**
- สินค้า SINGLE → เปลี่ยนเป็น `ACTIVATED`
- สินค้า PACK → นับ activation_count +1
- บันทึกข้อมูลผู้ Activate
- แสดงหน้ายืนยันสำเร็จ

## สถานะสินค้า (Product Status)

```
PENDING_LINK → IN_STOCK → PENDING_OUT → SHIPPED → ACTIVATED
     ↑              ↓             ↓                    ↓
     │          (ปฏิเสธ)      (ยกเลิก)             RETURNED
     │              ↓             ↓
   (Pre-Gen)    IN_STOCK      IN_STOCK

                              DAMAGED (แยกต่างหาก)
```

| สถานะ | คำอธิบาย |
|-------|----------|
| PENDING_LINK | QR สร้างล่วงหน้า รอเชื่อมโยง |
| IN_STOCK | อยู่ในคลัง พร้อมขาย |
| PENDING_OUT | รอการอนุมัติส่งออก |
| SHIPPED | ส่งออกแล้ว |
| ACTIVATED | ลูกค้าลงทะเบียนแล้ว |
| RETURNED | ถูกส่งคืนแล้ว |
| DAMAGED | เสียหาย |

---
