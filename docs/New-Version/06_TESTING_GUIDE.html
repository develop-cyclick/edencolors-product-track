วัตถุประสงค์: ทดสอบการสร้าง QR Code + Serial 12 หลักล่วงหน้า ก่อนรับสินค้าจริง

ขั้นตอนการทดสอบ

StepActionExpected Result
1Login ด้วย warehouse1 / warehouse123เข้า Dashboard สำเร็จ
2คลิกเมนู "สร้าง QR ล่วงหน้า"แสดงหน้า Pre-Generate
3กรอก จำนวน: 5-
4กรอก หมายเหตุ: Batch ทดสอบสำหรับ Filler-
5กดปุ่ม "สร้าง QR"สร้าง Batch สำเร็จ
ผลลัพธ์ที่คาดหวัง:
✅ สร้าง Batch เลขที่: PG-2026-000001
✅ สร้าง Serial 5 รายการ สถานะ PENDING_LINK

Data Flow

Warehouse User → PreGeneratedBatch (batch_no, quantity, created_by) → ProductItem (serial12, status=PENDING_LINK) → QRToken (token_hash, version=1, status=ACTIVE)

Test Case 6: ตรวจสอบความแท้ (Verify) PUBLIC

วัตถุประสงค์: ทดสอบการตรวจสอบความแท้ของสินค้าโดยไม่ต้อง Login

วิธีที่ 1: สแกน QR Code

ใช้มือถือสแกน QR Code → เปิดหน้า Verify อัตโนมัติ → URL: /v/123456789001

วิธีที่ 2: พิมพ์ Serial

เปิด /th/verify → กรอก Serial 12 หลัก → กดปุ่ม "ตรวจสอบ"

ผลลัพธ์ตามสถานะสินค้า

สถานะสินค้าสีข้อความปุ่ม Activate
IN_STOCK🟢 เขียวของแท้ - อยู่ในคลัง❌ ไม่แสดง
SHIPPED🟢 เขียวของแท้ - ส่งออกแล้ว✅ แสดง
ACTIVATED🟡 เหลืองของแท้ - ลงทะเบียนแล้ว❌ ไม่แสดง
RETURNED🟡 เหลืองของแท้ - ถูกส่งคืน❌ ไม่แสดง
Token Revoked🔴 แดงQR ถูกยกเลิก❌ ไม่แสดง
ไม่พบ🔴 แดงไม่พบในระบบ❌ ไม่แสดง

ผลลัพธ์ SINGLE:
✅ สินค้าเปลี่ยนสถานะ: SHIPPED → ACTIVATED
✅ สร้าง Activation record พร้อม consent_at, policy_version
✅ ปุ่ม "ลงทะเบียน" หายไป

7.2 Test Case: PACK Activation (หลายครั้ง)

เงื่อนไข: Product Master ต้องเป็น activation_type: PACK, max_activations: 3

Flow: ลงทะเบียนครั้งที่ 1 → activation_count: 0→1 → ยังกดได้ → ลงทะเบียนครั้งที่ 2,3 → ครบแล้วปุ่มหายไป

Test Case 11: ยืมสินค้า/สั่งฝาก (Loan/Reservation) ADMIN

15. Checklist การทดสอบ

Master Data Setup

  • มี ProductCategory อย่างน้อย 1 รายการ
  • มี Unit อย่างน้อย 1 รายการ
  • มี ShippingMethod อย่างน้อย 1 รายการ
  • มี Warehouse อย่างน้อย 1 รายการ
  • มี Clinic อย่างน้อย 1 รายการ
  • มี ProductMaster อย่างน้อย 1 รายการ
  • มี User ครบทุก Role (ADMIN, MANAGER, WAREHOUSE)

Functional Test

  • TC1: Pre-Generate QR สำเร็จ
  • TC2: สร้าง GRN + เชื่อมโยง Pre-Gen สำเร็จ
  • TC3: สร้าง Purchase Order สำเร็จ
  • TC4: สร้าง Outbound สำเร็จ
  • TC5a: Manager Approve สำเร็จ
  • TC5b: Manager Reject สำเร็จ
  • TC6: Verify (Public) แสดงผลถูกต้อง
  • TC7a: Activate (SINGLE) สำเร็จ
  • TC7b: Activate (PACK) หลายครั้งสำเร็จ
  • TC8: Reprint QR สำเร็จ + Token เดิม Revoked
  • TC9: Return สินค้าสำเร็จ
  • TC10: Mark Damaged สำเร็จ
  • TC11: Reservation flow ครบ

Security Test

  • Login ด้วย password ผิด → ถูก reject
  • เข้าหน้า Admin โดยไม่ login → redirect to login
  • WAREHOUSE เข้า Approval Board → 403 Forbidden
  • Rate Limit ทำงาน (60 req/min for public)

Edge Cases

  • Verify ด้วย Serial ที่ไม่มี → แสดง INVALID
  • Activate สินค้าที่ไม่ใช่ SHIPPED → Error
  • Reprint สินค้า ACTIVATED → Error
  • Return สินค้า IN_STOCK → Error
  • สแกน QR ที่ถูก Revoke → แสดง TOKEN_REVOKED
